## 彻底禁用浏览器访问的实施方案

### 背景
Wails v2 框架的开发模式无法完全禁用开发服务器，这是框架的核心功能。生产模式不启动开发服务器。

### 实施步骤

**步骤 1：构建生产版本**
- 使用 `wails build` 命令构建生产版本
- 生产版本不启动开发服务器，完全无法通过浏览器访问
- 将生产版本作为默认使用版本

**步骤 2：创建防火墙规则（可选，用于开发模式）**
- 创建 Windows 防火墙规则，阻止 Wails 开发服务器端口的入站连接
- 这样即使在开发模式下，也无法从浏览器访问
- 使用 PowerShell 命令创建防火墙规则

**步骤 3：更新项目文档**
- 更新 IFLOW.md，移除关于 HTTP 服务器和浏览器访问的错误描述
- 说明生产版本和开发版本的区别

### 预期效果
- 生产版本：完全无法通过浏览器访问
- 开发版本（有防火墙规则）：无法通过浏览器访问，但热重载功能正常
